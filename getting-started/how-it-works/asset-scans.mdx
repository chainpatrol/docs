## What is an Asset Scan

An **asset scan** is an automated security check ChainPatrol runs on an online surface that could be used in a scam, like a website, a social profile, an app-store listing, or a crypto address. Each scan records what we observed at a specific moment in time and evaluates risk using a mix of detection rules and AI signals.

<Info>
Asset scans solve a simple problem: the internet changes constantly, attackers iterate fast, and manual review doesn't scale. Scans let us monitor large volumes of assets, catch newly emerging threats, and track whether risky content is still live.
</Info>

## Asset vs. Asset Scan

Understanding the difference is key to how our monitoring works:

<Tabs>
  <Tab title="Asset" icon="link">
    An **asset** is the thing itselfâ€”a URL, handle, listing, or address.
    
    Example: `https://fake-metamask.com`
  </Tab>
  
  <Tab title="Asset Scan" icon="magnifying-glass">
    An **asset scan** is a **time-stamped snapshot** of that asset. The same asset may be scanned many times, and scans form a timeline of changes.
    
    Example: Scan #1 (Jan 1), Scan #2 (Jan 3), Scan #3 (Jan 5)
  </Tab>
</Tabs>

<Note>
Multiple scans of the same asset create a historical timeline, allowing us to track changes, detect when threats go live or are taken down, and monitor asset behavior over time.
</Note>

## When Scans Run

Scans are triggered in three main ways:

<AccordionGroup>
  <Accordion title="Automatically" icon="robot">
    Scans run automatically in several scenarios:
    
    - When ChainPatrol detection sources surface something relevant
    - When an asset is added (e.g., via report, onboarding, or API ingestion)
    - During **watchlist monitoring**, where assets are rescanned on a schedule that adapts over time:
      - More frequent early on
      - Less frequent as an asset stays stable
      - Blocked assets may be scanned less often to reduce noise
  </Accordion>
  
  <Accordion title="On Demand" icon="hand-pointer">
    Manual or API-triggered scans:
    
    - When a team member triggers a rescan from the UI
    - When your systems submit assets via API
  </Accordion>
  
  <Accordion title="Ongoing Liveness Checks" icon="heartbeat">
    For certain platform-hosted assets (e.g., specific site builders or marketplaces), ChainPatrol periodically re-checks whether a previously-blocked asset is still reachable.
  </Accordion>
</AccordionGroup>

## How Scans Work

Our scanning process follows a systematic approach to evaluate threats efficiently and accurately:

<Steps>
  <Step title="Normalize and Prepare">
    We normalize the input (e.g., resolve canonical URLs or identifiers), create a scan record, and run a first pass of checks that don't require external enrichment.
  </Step>
  
  <Step title="Enrich with Context">
    Depending on asset type, we collect relevant data:
    
    **Web URL / Page Enrichments:**
    - HTTP fetch results (status, redirects, extracted links, basic metadata like title/description)
    - Optional browser capture (screenshot/DOM signals when available)
    - Network and registration context (e.g., DNS, TLS, WHOIS/RDAP when enabled)
    
    **Platform/API Enrichments:**
    - Profile/listing details (descriptions, author/publisher fields, follower/engagement signals, etc.)
    
    <Tip>
    Some enrichments run only when earlier signals suggest elevated risk to keep scans efficient.
    </Tip>
  </Step>
  
  <Step title="Security Analysis">
    We evaluate the asset using rules and AI/ML signals for patterns like phishing, impersonation, and scam mechanics. This includes both content-based and visual/behavioral indicators when available.
  </Step>
  
  <Step title="Labels, Scores, and Liveness">
    Each scan is tagged with **labels** and produces **scores** for prioritization:
    
    **Common Labels:**
    - Captcha detected
    - Parked domain
    - Provider suspended
    - Brand impersonation
    - Wallet drainer detected
    
    **Liveness Status:**
    - **ALIVE** - Asset is active and accessible
    - **DEAD** - Asset is no longer reachable
    - **UNKNOWN** - Inconclusive (not "safe")
  </Step>
  
  <Step title="Relationship Discovery">
    For supported web and social surfaces, scans can extract outbound links and redirects to help you trace related infrastructure and uncover threat networks.
  </Step>
</Steps>
